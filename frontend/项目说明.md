# 项目说明文档

## 📖 项目简介

这是一个基于 **Vue 3 + Vite** 构建的前后端分离项目，实现了完整的 **RBAC（基于角色的访问控制）权限管理系统**。项目包含用户管理、动态菜单权限、按钮权限控制等核心功能。

---

### 前端框架

## ✨ 核心功能

### 1. 用户认证系统

#### 登录功能
- 用户名密码登录
- Token 存储（Cookie，有效期 7 天）
- 自动跳转（已登录访问登录页自动跳转到用户列表）

#### 注册功能
- 新用户注册
- 默认分配普通用户角色

#### 退出登录
- 清除所有认证信息
- 跳转到登录页

### 2. 动态菜单权限

#### 菜单结构
```
管理系统
├─ 👥 用户管理          ← 一级菜单（无子菜单）
└─ 🛠️ 系统管理 ›        ← 一级菜单（父菜单）
   └─ 🔐 角色管理       ← 二级菜单
```

#### 权限控制流程
1. 用户登录 → 服务端返回权限列表
2. 存储权限到 Cookie
3. 路由守卫拦截 → 动态添加异步路由
4. 侧边栏生成 → 只显示有权限的菜单
5. 访问路由 → 校验权限，无权限则重定向

#### 权限码说明
| 权限码 | 对应菜单/功能 | 说明 |
|--------|--------------|------|
| `user:view` | 用户管理 | 查看用户列表 |
| `user:create` | 新增用户按钮 | 创建新用户 |
| `user:update` | 编辑/角色按钮 | 更新用户信息 |
| `user:delete` | 删除按钮 | 删除用户 |
| `system:view` | 系统管理菜单 | 查看系统管理 |
| `role:view` | 角色管理 | 查看和配置角色 |

### 3. 按钮级权限控制

通过自定义指令 `v-permission` 实现：

```vue
<!-- 只有拥有 user:create 权限的用户才能看到 -->
<button v-permission="'user:create'" @click="addUser">
  新增用户
</button>

<!-- 只有管理员才能编辑 -->
<button v-if="isAdmin" v-permission="'user:update'" @click="startEdit">
  编辑
</button>
```

### 4. 用户管理功能

#### 用户列表
- 表格展示（ID、用户名、角色）
- 服务端分页（支持 5/10/20 条/页）
- 用户头像显示

#### 用户操作
- **新增用户** - 创建新用户（默认普通用户）
- **编辑用户** - 修改用户名
- **修改角色** - 调整用户角色（管理员/编辑/普通用户）
- **删除用户** - 删除指定用户

#### 角色管理
- 三种角色：管理员（admin）、编辑（editor）、普通用户（user）
- 不同角色显示不同颜色的徽章
- 仅管理员可修改角色

### 5. 多级菜单系统

#### 特性
- 支持无限级菜单嵌套
- 父菜单可展开/折叠
- 箭头图标指示展开状态
- 平滑的动画过渡
- 自动展开当前路由所在的父菜单
- 子菜单缩进显示层级

## 🔧 核心代码解析

### 1. 路由守卫与动态路由

```javascript
// src/router/index.js
router.beforeEach((to, from, next) => {
  const token = Cookies.get('token')
  
  // 登录/注册页面直接放行
  if (to.path === '/login' || to.path === '/register') {
    if (token) next('/users')
    else next()
    return
  }
  
  // 需要登录验证
  if (to.meta.requiresAuth && !token) {
    next('/login')
    return
  }
  
  // 动态添加异步路由
  if (token && !hasAddedAsyncRoutes) {
    const permissions = JSON.parse(
      Cookies.get('permissions', { withConverter: false })
    )
    addAsyncRoutes(permissions)
    next({ ...to, replace: true })
    return
  }
  
  // 权限校验
  const permissions = JSON.parse(
    Cookies.get('permissions', { withConverter: false })
  )
  if (!hasPermission(to, permissions)) {
    next('/users')
    return
  }
  
  next()
})
```

### 2. v-permission 指令

```javascript
// src/directives/permission.js
export default {
  mounted(el, binding) {
    const requiredPermission = binding.value
    const permissions = JSON.parse(
      Cookies.get('permissions', { withConverter: false }) || '[]'
    )
    const hasPermission = permissions.includes(requiredPermission)
    
    if (!hasPermission) {
      el.parentNode && el.parentNode.removeChild(el)
    }
  }
}
```

### 3. 菜单生成与过滤

```javascript
// src/utils/permission.js
export function filterAsyncRoutes(routes, permissions) {
  const res = []
  routes.forEach(route => {
    const tmp = { ...route }
    if (hasPermission(tmp, permissions)) {
      if (tmp.children) {
        tmp.children = filterAsyncRoutes(tmp.children, permissions)
      }
      res.push(tmp)
    }
  })
  return res
}

export function generateMenu(routes) {
  const menuList = []
  routes.forEach(route => {
    if (route.meta && route.meta.menuVisible !== false) {
      const menu = {
        path: route.path,
        name: route.name,
        title: route.meta.title || route.name,
        icon: route.meta.icon || '',
        redirect: route.redirect || null
      }
      if (route.children && route.children.length > 0) {
        menu.children = generateMenu(route.children)
      }
      menuList.push(menu)
    }
  })
  return menuList
}
```

### 4. 侧边栏多级菜单渲染

```vue
<!-- src/components/Sidebar.vue -->
<template v-for="item in menuList">
  <!-- 有子菜单 -->
  <div v-if="item.children" class="menu-group">
    <div @click="toggleMenu(item.path)">
      {{ item.title }}
      <span :class="{ expanded: isExpanded(item.path) }">›</span>
    </div>
    <transition name="slide">
      <div v-show="isExpanded(item.path)" class="submenu">
        <router-link
          v-for="child in item.children"
          :to="getFullPath(item.path, child.path)"
        >
          {{ child.title }}
        </router-link>
      </div>
    </transition>
  </div>
  
  <!-- 无子菜单 -->
  <router-link v-else :to="item.path">
    {{ item.title }}
  </router-link>
</template>
```

## 💡 最佳实践

### 1. Cookie 存储
```javascript
// ✅ 正确：设置 withConverter: false
Cookies.set('permissions', JSON.stringify(['user:view']), {
  expires: 7,
  withConverter: false  // 禁用 URL 编码
})

// ❌ 错误：可能导致中文乱码
Cookies.set('permissions', JSON.stringify(['user:view']))
```

### 2. 路由配置规范
```javascript
// ✅ 父菜单必须配置 redirect
{
  path: '/system',
  redirect: '/system/roles',  // 必须有
  children: [...]
}

// ✅ 子路由路径不要以 / 开头
{
  path: 'roles'  // 正确，会变成 /system/roles
}

// ❌ 错误示范
{
  path: '/roles'  // 错误，会变成 /system//roles
}
```

### 3. 权限码命名
- 使用 `模块：操作` 格式
- 统一小写，冒号分隔
- 语义清晰，如 `user:create`

## ⚠️ 注意事项

### 1. 路由刷新问题
动态添加的路由在页面刷新后会丢失，路由守卫会自动重新添加。

### 2. 权限变更
修改权限后需要重新登录或手动清除 Cookie：
```javascript
Cookies.remove('permissions')
location.reload()
```

### 3. 跨域问题
开发环境需要在 `vite.config.js` 配置代理：
```javascript
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true
      }
    }
  }
})
```

## 🐛 常见问题

### 1. 菜单不显示
- 检查权限码是否正确
- 检查 `menuVisible` 是否为 `true`
- 清除 Cookie 重新登录

### 2. 路由跳转 404
- 检查路由路径是否配置正确
- 确认是否已添加异步路由
- 检查父菜单的 `redirect` 配置

### 3. 按钮不显示
- 检查 `v-permission` 权限码是否正确
- 确认用户是否有该权限
- 查看控制台是否有错误
